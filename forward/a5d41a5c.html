2024年10月6日

参考文档：

  • go-gin
  • go-GORM

项目结构

需要实现的功能

实现一个图书管理系统

  1. 用户服务：（登录，注册）、token验证
  2. 书籍操作：（对书籍的增删改查操作）
├── controller // CLD：服务入口，负责处理路由，参数校验，请求转发
│   ├── book.go
│   └── user.go
├── dao // CLD：负责数据与存储相关功能（MySQL，Redis，ES等），以及业务的增删改查
│   ├── book.go
│   ├── mysql_init.go
│   └── user.go
├── go.mod
├── go.sum
├── main.go
├── middleware //中间件定义
│   └── auth.go
├── model // 模型定义
│   ├── book.go
│   └── user.go
└── router // 路由中间件注册
    └── router.go

创建数据库

create database books charset utf8mb4;

实现过程

  • 建立数据库连接
    • dao层GORM建立数据库连接
  • 封装用户和书籍模型
    • model层封装用户和书籍模型
  • 封装注册登陆、token验证、书籍的增删改查方法
    • dao层封装用户操作interface及方法
    • dao层封装书籍操作interface及方法
  • 实现用户登陆注册、书籍增删改查业务逻辑
    • controller层调用dao层实现用户注册登陆
    • controller层调用dao层实现书籍增删改查
  • 实现验签中间件
    • middleware层
  • 注册路由
    • router层注册controller层方法以及注册middleware层中间件
  • 启动
    • main
